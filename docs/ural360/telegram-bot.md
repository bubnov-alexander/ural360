# Telegram-бот `yst-travel`

## Назначение

Telegram-бот используется как рабочий сервис для заявок и взаимодействия с туристическими услугами Ural360. Он уже существует как Python-проект и был перенесен локально без директории `.venv`.

Оригинальный GitHub-репозиторий бота: <https://github.com/bubnov-alexander/yst-travel>.

Именно этот репозиторий является источником информации о Telegram-боте в документации Ural360. Локальная копия бота использовалась только для анализа структуры и сценариев; переносить `.venv`, секреты и локальные базы в основной backend-репозиторий не нужно.

## Что видно по структуре

В проекте есть:

- `main.py` - точка входа;
- `config.py` - конфигурация;
- `requirements.txt` - Python-зависимости;
- `app/handlers` - обработчики Telegram-сценариев;
- `app/keyboard.py` - кнопки и меню;
- `app/database` - модели, миграции и работа с хранилищем;
- `app/storage/database.db` - локальное SQLite-хранилище;
- `app/storage/ystTravelBot.service` - systemd unit для запуска сервиса;
- `app/utils/exsel.py` - экспорт и работа с табличными данными.

## Сущности бота

По структуре проекта видны отдельные сервисные сущности:

- услуги катамаранов;
- услуги трансфера;
- услуги SUP-досок;
- заявки и заказы;
- настройки и сценарии обработки;
- сортировка и вывод цен.

## Рекомендация

Не стоит переносить Telegram-бот на PHP сразу. Более надежный путь:

1. оставить Python-бот рабочим;
2. описать его текущие сценарии;
3. подключить его к общей базе только с ограниченными правами;
4. перенести бизнес-правила в Apiato;
5. заменить прямой доступ к БД на API-вызовы.

Такой подход снижает риск поломать уже работающую часть проекта и дает возможность переносить систему постепенно.
