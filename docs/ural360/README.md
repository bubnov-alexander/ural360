# Ural360: инфраструктура проекта

`Ural360` - проект для туристических услуг и маршрутов по Уралу: сплавы, прокат катамаранов, SUP-досок, трансфер, автостоянка и обработка заявок через сайт и Telegram-бота.

Эта папка описывает текущую инфраструктуру проекта и план переноса legacy-части на современный backend. Это не дамп продакшен-данных и не место для секретов. Здесь собраны архитектура, основные пакеты, зоны ответственности сервисов и дорожная карта миграции.

## Что есть сейчас

| Часть | Статус | Роль |
| --- | --- | --- |
| WordPress-сайт | Legacy | Публичный сайт, страницы, услуги, WooCommerce-каталог, галерея и контент |
| Telegram-бот [`yst-travel`](https://github.com/bubnov-alexander/yst-travel) | Рабочий Python-сервис | Прием и сопровождение заявок, работа с услугами, внутренний CRM-сценарий |
| Laravel Apiato backend | Целевая платформа | Новая API-архитектура, админка, управление сущностями и правами доступа |
| Filament | Установлен в Laravel-проект | Административная панель для управления контентом, услугами, заявками и медиа |

## Целевая идея

Проект постепенно переносится с WordPress на Laravel Apiato. Telegram-бот пока остается отдельным Python-сервисом, потому что он уже работает и содержит бизнес-логику. Первый прагматичный шаг - объединить данные на уровне общей базы, но дать боту доступ только к тем таблицам и операциям, которые ему реально нужны.

В дальнейшем бот можно перевести с прямой работы с БД на Apiato API, чтобы вся бизнес-логика и валидация находились в одном backend-приложении.

| Пакет | Для чего используется |
| --- | --- |
| `apiato/core` | Архитектура Laravel-приложения через Containers, Actions, Tasks, Requests и Transformers |
| `laravel/framework` | Базовый backend-фреймворк, в проекте используется Laravel 12 |
| `filament/filament` | Админ-панель для управления проектом |
| `filament/spatie-laravel-media-library-plugin` | Интеграция Spatie Media Library с Filament |
| `spatie/laravel-medialibrary` | Хранение и привязка изображений, галерей и файлов к моделям |
| `spatie/laravel-permission` | Роли и права, включая доступ администратора |
| `laravel/passport` | OAuth2/API-аутентификация для будущих внешних клиентов и бота |
| `wikimedia/composer-merge-plugin` | Подключение зависимостей из Apiato Ship и Containers |

## Доменные области

- услуги: катамараны, SUP-доски, трансфер, автостоянка;
- маршруты и сплавы по Чусовой;
- заявки, заказы и статусы обработки;
- клиенты и контактные данные;
- медиа: галереи, изображения услуг, файлы;
- пользователи админки, роли и права;
- интеграция Telegram-бота с ограниченным доступом к данным.

## Важные ограничения

- Секреты, `.env`, токены, дампы баз и приватные ключи не должны попадать в репозиторий.
- Миграциями и структурой БД должен владеть Laravel/Apiato backend.
- Telegram-бот на первом этапе может иметь доступ к общей БД, но только к разрешенным сущностям.
- Для бота нужен отдельный DB-пользователь без прав на изменение схемы.
