# Архитектура Ural360

## Общая картина

Ural360 состоит из трех логических частей:

| Компонент | Технология | Назначение |
| --- | --- | --- |
| Публичный сайт | WordPress + кастомная тема + WooCommerce | Контент, услуги, маршруты, галерея, страницы для клиентов |
| Telegram-бот | Python, aiogram-подход, локальная структура `yst-travel` | Обработка заявок и сервисный сценарий внутри Telegram |
| Новый backend | Laravel 12 + Apiato 13 + Filament | Централизованная бизнес-логика, админка, API и права доступа |

## Почему не переписывать все сразу

Telegram-бот уже решает рабочую задачу и содержит прикладную бизнес-логику. Переносить его на PHP только ради единого языка невыгодно: это увеличит риск поломать текущий процесс и не даст мгновенной пользы.

Более устойчивый путь:

1. оставить Python-бот как отдельный сервис;
2. вынести общие данные в единую БД;
3. ограничить права бота на уровне БД;
4. постепенно переносить бизнес-операции в Apiato;
5. после этого перевести бота на API вместо прямой работы с таблицами.

## Целевая схема

```text
Клиент
  |
  |-- Публичный сайт / будущий frontend
  |       |
  |       v
  |   Apiato API  <---->  MySQL / MariaDB
  |       ^                    ^
  |       |                    |
  |   Filament admin           |
  |                            |
  |-- Telegram bot ------------|
       сначала ограниченный DB-доступ,
       затем переход на Apiato API
```

## Ответственность сервисов

### Apiato backend

- владеет схемой базы данных и миграциями;
- содержит основную бизнес-логику;
- предоставляет API для внешних клиентов;
- управляет пользователями, ролями и правами;
- хранит медиа через Spatie Media Library;
- дает администраторам интерфейс через Filament.

### Filament admin

- управление услугами и маршрутами;
- управление заявками и статусами;
- работа с медиа и галереями;
- управление пользователями админки;
- операционная панель для проекта.

### Telegram-бот

- остается Python-сервисом;
- работает с заявками и сервисными сценариями;
- на первом этапе получает доступ только к нужным таблицам;
- не управляет миграциями и структурой БД;
- в будущем должен перейти на Apiato API.

### WordPress legacy

- источник текущей структуры сайта и контента;
- содержит кастомную тему, WooCommerce-интеграции, страницы, галерею и услуги;
- постепенно заменяется Laravel/Apiato-частью.
