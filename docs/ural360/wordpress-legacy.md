# WordPress legacy

## Роль текущего сайта

WordPress-сайт `10451.grampus-server.ru` является текущей публичной частью проекта. Он содержит контент, услуги, галерею, маршруты и WooCommerce-структуру.

## Что найдено в структуре

В кастомной теме есть шаблоны:

- `front-page.php`;
- `archive-gallery.php`;
- `archive-news.php`;
- `archive-questions.php`;
- `archive-service.php`;
- WooCommerce overrides;
- кастомные хуки WooCommerce в `inc/wc_hooks.php`.

По структуре и ссылкам видны направления:

- прокат катамаранов;
- трансфер;
- автостоянка;
- прокат SUP-досок;
- сплавы по реке Чусовой;
- маршруты и точки вроде Усть-Утки, Ёквы и Сулёма;
- галерея и информационные страницы.

## Что нужно перенести

| Данные | Куда переносить |
| --- | --- |
| Услуги | Apiato Container `Service` |
| Маршруты и сплавы | Apiato Container `Route` |
| Галерея и изображения | Spatie Media Library |
| Заявки и заказы | Apiato Container `Booking` |
| Страницы и SEO | Новый frontend/API-модель страниц |
| WooCommerce-товары | Проверить: часть станет услугами или маршрутами |

## Риски миграции

- URL и slug могут быть важны для SEO;
- изображения надо переносить вместе с привязками к сущностям;
- WooCommerce может содержать данные, которые не являются обычными товарами;
- часть логики может быть спрятана в теме и хуках;
- нужно сохранить рабочие клиентские сценарии на время перехода.

## Подход

WordPress не надо отключать первым шагом. Его лучше использовать как источник данных и постепенно переносить сущности в Apiato. После того как новые API, админка и публичный frontend будут готовы, можно переключать сайт.
